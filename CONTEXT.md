# 프로젝트 컨텍스트

## 서비스 한 줄 정의
공공데이터 + AI를 활용해 소상공인·세입자·투자자에게 부동산 시장의 숨겨진 정보를 쉽게 전달하는 데이터 인텔리전스 플랫폼

---

## 문제 인식

### 한국 부동산 시장의 구조적 문제
- 건물주는 임대료보다 **시세 차익**이 목적 → 콧대 높고 협상 동기 없음
- 건물주가 중개사에 모든 것을 일임 → 정보 독점 구조 고착화
- 공공데이터는 존재하지만 일반인이 해석하기 어려움
- 소상공인은 젠트리피케이션 위험을 창업 전에 알 방법이 없음
- 세입자는 계약 전 건물·중개사의 문제 이력을 확인할 수 없음

### 기존 서비스의 한계
- 네이버 부동산, 호갱노노: 데이터 시각화만 있고 **해석이 없음**
- "그래서 나 어떻게 해야 해?" 라는 질문에 아무도 답 안 해줌

---

## 핵심 인사이트

> 데이터를 보여주는 게 아니라, **결론을 말해주는 서비스**

AI 이전엔 컨설턴트를 고용해야 알 수 있었던 것들을 무료/저가에 제공

---

## 타겟 사용자 (을의 관점)

| 타겟 | 페인포인트 | 원하는 것 |
|------|-----------|----------|
| 소상공인 | 창업 후 임대료 폭등·젠트리피케이션 | "여기 창업해도 돼요?" |
| 세입자 | 전세사기, 계약 후 분쟁 | "이 집 계약해도 안전해요?" |
| 예비 투자자 | 정보 없이 부동산 시장 진입 | "다음에 오를 동네가 어디야?" |

---

## 서비스 구조 (2가지 레이어)

### Layer A — 을 보호 서비스 (트래픽 확보)
- 상권 임대료 위험도 조회
- 젠트리피케이션 위험 등급
- 계약 안전 분석 (등기부 자동 분석)
- 무료 또는 저가

### Layer B — 투자 인텔리전스 (수익원)
- 부동산 가치 상승 선행 신호 감지
- "다음 성수동은 어디인가"
- 월 구독 유료 서비스

---

## 부동산 가치 상승 선행 신호 체계

```
3~5년 전: 지하철 노선 계획, 재개발 구역 지정, 도시기본계획 변경
1~2년 전: 젊은 층 인구 유입, 소규모 리모델링 허가 증가
6~12개월 전: 특정 업종 인허가 급증(카페·갤러리), 공실률 감소 시작
```

## 데이터 소스 (모두 공공데이터)

- 국토교통부 실거래가 API
- 국토교통부 상가임대차 분쟁조정 데이터
- 소상공인시장진흥공단 상권분석 데이터
- 건축인허가 데이터 (세움터)
- 등기부등본 (대법원 인터넷등기소)
- 행정안전부 주민등록 인구통계 (전입·전출)
- 도시계획 인허가 정보

## MVP 범위 (1단계)

### 목표
서울 주요 상권 20개 커버하는 상권 위험도 분석 서비스

### 핵심 기능
1. 지역명/주소 입력
2. 공공데이터 조회 (임대료 추이, 분쟁 이력, 공실률, 업종 변화)
3. LLM 종합 분석
4. 자연어 리포트 출력

### MVP 예시 출력
```
"홍대 인근 현재 임대료 상승률 연 8.3%,
최근 3년간 분쟁 건수 증가 추세.
5년 운영 목표라면 권리금 손실 위험이 있습니다.
연남동 쪽이 상대적으로 안정적입니다."
```

## 기술 스택 (제안)

- Backend: Python (FastAPI)
- Database: PostgreSQL
- AI: Claude API (분석 및 자연어 리포트 생성)
- 공공데이터: 각 기관 Open API
- Frontend: Next.js (추후)
