# MVP 실행 기획안

> 이 문서는 [CONTEXT.md](CONTEXT.md)의 비전을 기반으로, Sprint 0(공공데이터 API 검증) 결과를 반영하여 MVP를 재정의한 실행 계획서입니다.

---

## 1. 핵심 비전 (CONTEXT.md 요약)

> **데이터를 보여주는 게 아니라, 결론을 말해주는 서비스.**

공공데이터 + AI로 **소상공인·세입자·투자자**에게 부동산/상권의 숨겨진 정보를 해석해 전달하는 데이터 인텔리전스 플랫폼.

- **Layer A (을 보호)**: 상권 위험도·계약 안전 분석 — 무료/저가, 트래픽 확보
- **Layer B (투자 인텔리전스)**: 가치 상승 선행 신호 감지 — 월 구독, 수익원

## 2. Sprint 0 결과: 데이터 가용성 검증

4개 공공데이터 API를 승인받아 실제 호출·데이터 구조·커버리지를 검증했다.

### 검증 완료된 API

| API | 출처 | 상태 | 주요 데이터 |
|---|---|:-:|---|
| 상업업무용 부동산 매매 실거래가 | 국토교통부 (data.go.kr) | 정상 | 상가 매매가, 면적, 건물용도, 거래일 |
| 상가(상권)정보 | 소상공인시장진흥공단 (data.go.kr) | 정상 | 점포 위치·업종·좌표 (서교동 8,829개 등) |
| 행정동별 인구·세대 | 행정안전부 (data.go.kr) | **미완** | 엔드포인트 확인됐으나 파라미터 매칭 실패 |
| 상권분석(추정매출·영역) | 서울 열린데이터광장 | 정상 | 행정동 단위 업종별 매출, 상권 경계 |

### 서울 열린데이터광장 상세

| 서비스 | 서비스명 | 용도 |
|---|---|---|
| 서울시 전체 매출 | `VwsmMegaSelngW` | 전국 비교용 |
| 자치구 단위 매출 | `VwsmSignguSelngW` | 구 단위 집계 |
| **행정동 단위 매출** | **`VwsmAdstrdSelngW`** | **MVP 핵심** |
| 상권 영역(경계) | `TbgisTrdarRelm` | 1,650개 상권 폴리곤 |

### 매출 API 확인된 분해 축 (MVP 분석의 무기)

- **시계열**: 2019 Q1 ~ 2025 Q4 (분기 단위, 최신 2025 Q4)
- **업종**: 100개 생활밀접업종 (외식 10/서비스 47/소매 43)
- **시간축**: 요일별·시간대별(00~06, 06~11, 11~14, 14~17, 17~21, 21~24)
- **고객축**: 성별·연령대(10대~60대+)
- **매출/건수** 양쪽 모두 제공 → 객단가 계산 가능

## 3. 데이터 가용성 매트릭스

CONTEXT.md의 MVP 기능별 현 상태를 정리하면:

### 커버되는 영역 (MVP v1에 포함)

| 기능 | 커버리지 | 활용 데이터 |
|---|:-:|---|
| 상권 업종 구조 분석 | 완벽 | 소상공인진흥공단 + 서울 매출 |
| 매출 기반 수익성 판단 | 완벽 | 서울 추정매출 8분기 시계열 |
| 업종별 성장/하락 트렌드 | 완벽 | 분기별 매출 증감률 |
| 시간대/요일/성별/연령 분해 | 완벽 | 매출 API 세부 필드 |
| 동 간 비교 | 완벽 | 행정동 단위 매출 |
| 상권 경계/위치 시각화 | 완벽 | `TbgisTrdarRelm` + shapefile |

### 문제 영역 (MVP v2 이후 또는 축소)

| 기능 | 상태 | 문제 |
|---|:-:|---|
| 임대료 상승률("연 X%") | 불가 | 매매가만 있음, 상가 전월세 데이터 없음 |
| 임대료 부담률(임대료/매출) | 불가 | 임대료 데이터 없음 |
| 계약 안전 분석(등기부) | 불가 | 유료 데이터, 공공 API 없음 |
| 임대차 분쟁 건수 추이 | 불가 | 승인 API에 포함 안 됨 |
| 재개발/지하철 선행지표 | 미확보 | Layer B용 별도 API 필요 |

### 검증된 실사례 (서교동/홍대 2024 Q3 → 2025 Q4)

- **성장 업종**: 반찬가게 +132%, 양식 +34%, 중식 +31%, 일식 +30%, 한식 +16%
- **하락 업종**: 서적 -43%, 핸드폰 -37%, 화장품 -24%, **카페 -12%, 주점 -11%**
- **구조 변화**: "술·카페 핫플"에서 **"식사 중심 상권"**으로 재편 중
- **고객**: 20~30대가 한식 매출의 66% 차지

→ 이 수준의 인사이트를 네이버 부동산·호갱노노는 제공하지 못한다.

## 4. MVP v1 재정의 (이번 스프린트 목표)

### 서비스명 (가안)
**상권 트렌드 리포트** — "여기 창업해도 돼요?"에 답하는 서비스

### 기능 범위

1. **입력**: 서울시 행정동(또는 상권) 선택
2. **처리**:
   - 해당 동의 매출 추이 (최근 8분기)
   - 업종 분포 + 업종별 성장/하락률
   - 시간대·요일·성별·연령 고객 분해
   - 주변 동과 비교 (인접 3~5개 동)
   - 젠트리피케이션 위험 스코어 (프랜차이즈·주점·카페 비율 기반)
3. **출력**: LLM(Claude API) 기반 자연어 리포트
4. **타겟**: 서울시 주요 상권 20개 (서교동·연남동·성수·강남·가로수길 등)

### 제외되는 기능 (v1에서는)

- 임대료 분석 (임대료 API 확보 후)
- 계약 안전 분석 (등기부) — 스코프 밖
- 부동산 투자 선행 지표 (Layer B) — v2 이후

## 5. 로드맵

### Phase 1 (v1): 상권 트렌드 리포트 — 현재 집중
**목표**: 기존 데이터로 만들 수 있는 최대치를 만든다.

- 데이터 수집 파이프라인 (서울 매출 + 상가정보 + 실거래가)
- PostgreSQL DB 스키마 + 분기별 적재
- LLM 분석 프롬프트 설계
- FastAPI 백엔드 + 리포트 생성 API
- 간단한 프론트엔드 (Next.js)

### Phase 2 (v1.5): 임대료 연동
**목표**: CONTEXT.md 원안 완성.

- 국토교통부 **상업용 전월세 실거래가** API 추가 신청
- 임대료 부담률(임대료/매출) 분석 추가
- "임대료 상승률 연 X%" 계산 구현

### Phase 3 (v2): 투자 인텔리전스(Layer B)
**목표**: "다음 성수동" 예측.

- 인구 전입/전출 (행정안전부)
- 건축인허가 (세움터)
- 재개발·지하철 계획 (서울시 도시계획)
- 선행지표 모델 학습

### Phase 4 (v2.5): 수익화
- 유료 구독 전환 (월 리포트, 기업용 API)
- 계약 안전 분석 (등기부 스크래핑 검토)

## 8. 해자(Moat) 관점

CONTEXT.md의 "진짜 해자: 시간이 쌓일수록 강해지는 데이터"를 실현하려면:

- 서울 매출 API는 2019년부터 분기별 제공 → **과거 7년치 시계열 확보 가능**
- 분기별 자동 수집 파이프라인을 지금 만들어두면, **1년 후 내 DB에는 28분기치 누적**
- 공공 API 자체가 아닌 **정제·결합·해석 레이어**가 해자가 됨
- 사용자 행동 데이터 + 예측 모델 + 브랜드 신뢰가 진짜 해자

→ v1 출시보다 **데이터 축적 파이프라인이 먼저**라는 결론.
